## Standalone Vulnerabilities

### VULN-01: A01 — Broken Access Control

- **Severity:** Medium
- **Location:** `src/index.js`:139-155 (method: `GET /api/activities/:id`)
- **CWE:** [CWE-639](https://cwe.mitre.org/data/definitions/639.html)

#### Description
Viewing activity logs by ID lacks user ownership validation, allowing any authenticated runner to retrieve another customer's private fitness details.

### VULN-02: A06 — Vulnerable and Outdated Components

- **Severity:** Medium
- **Location:** `src/index.js`:157-170 (method: `unsafeMerge`)
- **CWE:** [CWE-1321](https://cwe.mitre.org/data/definitions/1321.html)

#### Description
An insecure recursive object merger helper fails to check for prototype properties, exposing the application to prototype pollution.

### VULN-03: A07 — Identification and Authentication Failures

- **Severity:** Medium
- **Location:** `src/index.js`:96-112 (method: `POST /api/auth/login`)
- **CWE:** [CWE-330](https://cwe.mitre.org/data/definitions/330.html)

#### Description
Session identifiers are created using predictable Math.random() values instead of secure PRNG tokens.

## Chained Attack Scenarios

Chained scenarios represent multiple code-level weaknesses that, when exploited in sequence, lead to high-impact outcomes.

### CHAIN-01: Predictable Session Hijacking → IDOR Fitness Log Theft

- **Combined Impact:** `db_exfiltration`
- **Difficulty:** Hard
- **Subtlety Tags:** `cross_file` `stateful_flow` `implicit_trust` `realistic_decoy`

#### Prerequisites
- attacker has or can create a low privilege account
- attacker can combine request input with stored application state

#### Attack Narrative
An attacker guesses active session keys generated by Math.random(), sets the session cookie in their browser, and calls `/api/activities/1` to view private fitness workouts of other users, achieving database exfiltration.

#### Chain Components
| Step | Description | Severity | OWASP | CWE | Location | Method |
|---|---|---|---|---|---|---|
| 1 | Weak session token generation via predictable random number generator. | Medium | A07 | CWE-330 | src/index.js | `POST /api/auth/login` |
| 2 | Activities endpoint is vulnerable to IDOR fetching. | Medium | A01 | CWE-639 | src/index.js | `GET /api/activities/:id` |

### CHAIN-02: Subtle Auth Session Pivot To Idor

- **Combined Impact:** `db_exfiltration`
- **Difficulty:** Hard
- **Subtlety Tags:** `cross_file` `stateful_flow` `implicit_trust` `realistic_decoy` `secondary_chain`

#### Prerequisites
- attacker has or can create a low privilege account
- attacker can combine request input with stored application state

#### Attack Narrative
Attacker combines a low-visibility entry point with stored or derived application state, then pivots to a higher-impact sink that is reachable only after following the cross-file flow.

#### Chain Components
| Step | Description | Severity | OWASP | CWE | Location | Method |
|---|---|---|---|---|---|---|
| 1 | Session identifiers are created using predictable Math.random() values instead of secure PRNG tokens. | Medium | A07 | CWE-330 | src/index.js | `POST /api/auth/login` |
| 2 | An insecure recursive object merger helper fails to check for prototype properties, exposing the application to prototype pollution. | Medium | A06 | CWE-1321 | src/index.js | `unsafeMerge` |
| 3 | Viewing activity logs by ID lacks user ownership validation, allowing any authenticated runner to retrieve another customer's private fitness details. | Medium | A01 | CWE-639 | src/index.js | `GET /api/activities/:id` |

## Standalone Vulnerabilities

### VULN-01: A01 — Broken Access Control

- **Severity:** Medium
- **Location:** `src/index.js`:164-175 (method: `GET /api/events/:id/guests`)
- **CWE:** [CWE-639](https://cwe.mitre.org/data/definitions/639.html)

#### Description
Viewing event guest lists by event ID lacks verification of user ownership, allowing any authenticated planner to view contact details of other event attendees.

### VULN-02: A02 — Cryptographic Failures

- **Severity:** Medium
- **Location:** `src/index.js`:116-136 (method: `POST /api/auth/login`)
- **CWE:** [CWE-328](https://cwe.mitre.org/data/definitions/328.html)

#### Description
User account passwords are encrypted using unsalted MD5 hashing and saved in the database, allowing brute-force or rainbow table cracking.

### VULN-03: A07 — Identification and Authentication Failures

- **Severity:** Medium
- **Location:** `src/index.js`:116-136 (method: `POST /api/auth/login`)
- **CWE:** [CWE-330](https://cwe.mitre.org/data/definitions/330.html)

#### Description
Session identifiers are created using predictable Math.random() values instead of secure PRNG tokens.

## Chained Attack Scenarios

Chained scenarios represent multiple code-level weaknesses that, when exploited in sequence, lead to high-impact outcomes.

### CHAIN-01: Predictable Session Hijacking → IDOR Guest List Harvesting

- **Combined Impact:** `db_exfiltration`
- **Difficulty:** Hard
- **Subtlety Tags:** `cross_file` `stateful_flow` `implicit_trust` `realistic_decoy`

#### Prerequisites
- attacker has or can create a low privilege account
- attacker can combine request input with stored application state

#### Attack Narrative
An attacker guesses active session keys generated by Math.random() belonging to a wedding coordinator, sets the session cookie in their browser, and calls `/api/events/2/guests` to retrieve guest lists (including names and emails) of other users via IDOR.

#### Chain Components
| Step | Description | Severity | OWASP | CWE | Location | Method |
|---|---|---|---|---|---|---|
| 1 | Weak session token generation via predictable random number generator. | Medium | A07 | CWE-330 | src/index.js | `POST /api/auth/login` |
| 2 | Guests endpoint is vulnerable to IDOR fetching. | Medium | A01 | CWE-639 | src/index.js | `GET /api/events/:id/guests` |

### CHAIN-02: Subtle Auth Session Pivot To Idor

- **Combined Impact:** `db_exfiltration`
- **Difficulty:** Hard
- **Subtlety Tags:** `cross_file` `stateful_flow` `implicit_trust` `realistic_decoy` `secondary_chain`

#### Prerequisites
- attacker has or can create a low privilege account
- attacker can combine request input with stored application state

#### Attack Narrative
Attacker combines a low-visibility entry point with stored or derived application state, then pivots to a higher-impact sink that is reachable only after following the cross-file flow.

#### Chain Components
| Step | Description | Severity | OWASP | CWE | Location | Method |
|---|---|---|---|---|---|---|
| 1 | Session identifiers are created using predictable Math.random() values instead of secure PRNG tokens. | Medium | A07 | CWE-330 | src/index.js | `POST /api/auth/login` |
| 2 | User account passwords are encrypted using unsalted MD5 hashing and saved in the database, allowing brute-force or rainbow table cracking. | Medium | A02 | CWE-328 | src/index.js | `POST /api/auth/login` |
| 3 | Viewing event guest lists by event ID lacks verification of user ownership, allowing any authenticated planner to view contact details of other event attendees. | Medium | A01 | CWE-639 | src/index.js | `GET /api/events/:id/guests` |
